The premise · SD is a different room, same house

LLD's wing feels like a pattern library: small, intimate, serif-typeset, amber-lit, one concept per room. System Design needs a different spatial feeling — large drafting hall, open canvas, blueprints on walls, wind-tunnel visible through a tall window. Same house, different vibe. This batch defines the extension.

📐
THE SAME-HOUSE PRINCIPLE
"A user who learned to love the LLD wing should enter the SD hall and know — without being told — that they are still in Architex."
The visual language transfers (serif typography, ambient light, earned color). The spatial feel shifts (bigger canvas, more air, blueprints over walls). Cross-module pointers work in one click. Seamless means: no re-learning the chrome, but instant recognition of the new room's purpose.

Q29 · How different should SD look from LLD?

Too identical = no signal. Too different = it's a separate product.

OPTION A · IDENTICAL
Same chrome. Same color. Same type. Only content differs. Maximum cohesion but zero room-signaling.
⭐ OPTION B · ACCENT SHIFT
Same layout, same type, same motion language. One accent color swaps: LLD amber/rust → SD cobalt/teal. Subtle but immediate. Like walking from a warm library to a cool drafting hall.
━━━ LLD accent: warm amber #fb923c
━━━ SD accent: cool cobalt #60a5fa
Background, type, motion, spacing — identical. Accent = signal only.
OPTION C · ROOM-ARCHITECTURE
Accent shifts plus layout density shifts. LLD: dense 3-column reading layout. SD: airier 2-column with large canvas. Different proportions for different work.
OPTION D · FULL ROOM CHARACTER
Accent + layout + micro-interactions. SD has "blueprint-grid" background on canvas, subtle blue-white ambient light, drafting-ruler tick-marks on panels. Strong character, more build effort.

Q30 · Panel architecture — how do chaos / cost / coach / trace coexist?

Simulate mode especially has many surfaces competing for screen. Architecture matters.

OPTION A · FIXED THREE-COLUMN
Left: problem/sidebar. Center: canvas. Right: metrics+coach+trace stacked. Predictable. Rigid when canvas wants more space.
⭐ OPTION B · COLLAPSIBLE DOCKED
3-column baseline, each side collapsible to 40px rail. User presses [ / ] to expand/collapse sides. Canvas claims freed space. Panels inside right column stack or tab.
▸ problem │ [ canvas ] │ metrics
           │             │ coach
           │             │ trace
           │             │ cost
OPTION C · FLOATING WINDOWS
Panels are draggable/resizable windows over full-canvas backdrop. User arranges personal workspace. Powerful but busy; good for power users.
OPTION D · COMMAND BAR + OVERLAYS
Canvas full-screen always. Single command bar at bottom (⌘K). All info summoned as transient overlays. Minimal, cinema-grade.

Q31 · Switching between 5 modes (Learn / Build / Simulate / Drill / Review)

The mode switcher is one of the most-pressed UI element. Cross-module consistency matters since LLD also has modes.

OPTION A · TOP TAB BAR
5 tabs across the top. Always visible. Click to switch. Takes 40px of vertical real-estate permanently.
⭐ OPTION B · LEFT RAIL ICONS + ⌘1-5
Thin 48px left rail with 5 mode icons. Current mode highlighted. Keyboard: ⌘1 Learn, ⌘2 Build, ⌘3 Simulate, ⌘4 Drill, ⌘5 Review. Matches LLD's rail (which uses 1-4). User muscle memory transfers.
│ 📖 │ ← Learn (⌘1)
│ 🎨 │ ← Build (⌘2)
│ ⚡ │ ← Simulate (⌘3) · active
│ ⏱ │ ← Drill (⌘4)
│ 🔁 │ ← Review (⌘5)
OPTION C · COMMAND PALETTE ONLY
No visible mode switcher. ⌘K opens palette: "switch to simulate". Minimal. High-friction for discovery.
OPTION D · CONTEXTUAL CROSSFADE
Mode switcher hidden in a floating circle bottom-right. Hovering expands petals (radial menu). Dramatic, showpiece, but unfamiliar.

Q32 · Concept vs Problem page — what formats ship?

From Batch 3: 40 concepts (Learn atoms) + 30 problems (Drill molecules). Each needs a page format. Multi-select.

① 8-SECTION CONCEPT PAGE ⭐
Matches LLD's 8-section Learn format. Sections: The Itch · Definition · Mechanism · Anatomy (diagram) · Scaling Numbers · Real-World Uses · Failure Modes · Checkpoints. 1200-1800 words. Cross-linked.
② PROBLEM PAGE · 6-PANE LAYOUT ⭐
Panes: Problem statement · Functional & non-functional requirements · Scale numbers · Canonical design (diagram + code) · Failure modes + chaos scenarios · Concepts used. 2500-3500 words.
③ SCALING NUMBERS STRIP ⭐
Every concept + problem pins a "napkin numbers" strip at top: typical QPS · storage · cost per user · p99 · common bottleneck. Glanceable. First thing a senior looks at.
④ DECISION TREE VISUAL ⭐
For concepts with branching choices (caching strategies, replication modes): an interactive decision tree. "When to use X vs Y vs Z?" Click a leaf to see the concrete rule.
⑤ ENGINEERING BLOG DEEP-LINKS
Each page cites 3-5 real engineering blog posts (Netflix, Uber, Discord, High Scalability, Increment). Authoritative references, curated.
⑥ ASK-AI INLINE
Highlight any paragraph → "explain to me" popover with Haiku. "Why is this?" / "Give me an example." / "How does this compare to X?" Reading aid.
⑦ "TRY IT" BUTTON ⭐
Every concept page has a "Try it in Simulate" button. Every problem page has "Drill this" and "Simulate canonical". The Learn→Act loop from Batch 4.
⑧ CROSS-MODULE BRIDGE CARD
At end of each concept / problem: "Related LLD patterns" card with thumbnails + 1-line hooks. Enables seamless traversal across modules.

Q33 · Animation & motion philosophy

Motion timing shapes how a UI feels. LLD uses serif-timed ease (550ms, deliberate). Does SD match or diverge?

OPTION A · MATCH LLD EXACTLY
550ms ease-out for meaningful transitions, 180ms for state flicks, serif-timed elastic for opening panels. Same library, same tokens.
⭐ OPTION B · MATCH + FAILURE CINEMATICS
All baseline animations match LLD. But chaos events get their own motion vocabulary: 900ms ribbon slide, 2.4s pulse halo, edge-drip propagation. Normal UI = calm. Failure = cinema. Signal through contrast.
OPTION C · FASTER, DATA-DENSE
Because SD screens have more data, motion is 30% faster (300ms vs 550). Feels snappier, more responsive. Diverges from LLD serif-timing.
OPTION D · MINIMAL MOTION
Only functional animations. No chrome flourishes. Respects "reduce motion" systemically. For accessibility-first users.

Q34 · First-time mode entry — what does the user see?

User enters Simulate for the very first time. Canvas is blank. Panels ready. What frames the moment?

OPTION A · EMPTY CANVAS + DOCS
Empty state with "load a design or pick a problem" buttons + link to help docs. Fastest to build. Weakest onboarding.
⭐ OPTION B · GUIDED 90-SEC SPOTLIGHT
Short animated walkthrough with dimmed spotlight over each panel + 1-sentence purpose. 90 seconds end-to-end. Dismissible. Remembered per user. Serif type, earned motion.
"Canvas — your design lives here."
"Metrics — your design breathing in real time."
"Chaos panel — the storms you can summon."
"Coach — whispers only when it matters."
"Press space to pause time. You're ready."
OPTION C · SAMPLE RUN
Auto-plays a 2-minute pre-recorded simulation of "Design Netflix" → user watches once, all UI elements live. Then canvas clears. Learn by watching.
OPTION D · SCENARIO LIBRARY LANDING
First entry opens a curated grid of scenarios: "Validate my design", "Try chaos drill", "Replay Facebook 2021". User picks a starting point; workspace forms around that choice.
