The premise · Box-and-arrow is the default, not the ceiling

So far we've assumed ReactFlow-style box-and-arrow diagrams because that's what the existing canvas is. But System Design is a multi-lens discipline: architecture diagrams, sequence diagrams, deployment topologies, data flow maps, state machines. And the same architecture can be viewed through overlays: latency heat, cost heat, blast radius, critical path. This batch expands what a diagram can be, what components it contains, and what learning formats surround it.

Q47 · Diagram types — which to ship beyond architecture boxes?

Each diagram type teaches a different axis. Multi-select. More = more surface but richer pedagogy.

① ARCHITECTURE DIAGRAM ⭐ baseline
what we have today
Boxes = services/stores. Arrows = calls. The canvas we already have. Keep.
② SEQUENCE DIAGRAM ⭐
Time runs top-to-bottom. Shows request flow across services: "client → gateway → auth → app → cache → db → back". Critical for showing how a request travels. Interview-standard.
client ─────▶ gateway ─────▶ auth
   │         │        │◀────── token
   │         │─────▶ app ─────▶ cache (miss)
   │         │           │─────▶ db
③ DEPLOYMENT TOPOLOGY ⭐
Regions → AZs → hosts → containers → services. Shows the physical placement. Reveals blast radius of zone/region outage at a glance. Chaos drill clicks through it.
④ DATA FLOW DIAGRAM ⭐
Shows where data moves + transforms. Ingestion → enrichment → storage → serving. Critical for streaming systems and ETL-heavy problems.
⑤ STATE MACHINE
For protocols (Paxos, Raft, 2PC, Saga). States as circles, transitions as arrows. Critical for teaching consensus + transactions.
⑥ ER / DATA MODEL
Tables / collections / entities + relationships. Often a sub-panel of the main diagram but sometimes its own view.
⑦ NETWORK TOPOLOGY
VPCs, subnets, routes, LBs, firewalls, peering. For enterprise problems + infra deep-dives.
⑧ SWIMLANES / OWNERSHIP MAP
Team-by-team column layout. "Who owns what?" Shows org/system coupling (Conway's Law made visible). Unique moat — no competitor ships this.
⑨ SERVICE MESH / DEPENDENCY GRAPH
Force-directed. Shows the real call graph at scale. Great for "find the single point of failure" drills.
⑩ CASCADE / BLAST-RADIUS GRAPH
Click any node → all downstream nodes that would fail if this dies. Live highlighting. Pairs with chaos events. Our moat in failure pedagogy.

Q48 · SD node-type library — what components live on the canvas?

LLD has class-box nodes. SD needs richer, more distinct types. Multi-select the families to ship.

① CLIENT TIER ⭐
Web browser · mobile · IoT · admin · CLI · bot. Icons distinguish. Entry-point for request traces.
② EDGE & CDN ⭐
CloudFront / Cloudflare / Fastly · DNS · Anycast · edge compute · WAF.
③ LOAD BALANCERS ⭐
L4 TCP/UDP · L7 HTTP · round-robin · consistent-hash · weighted · sticky-session. Visual distinguishes.
④ API GATEWAYS ⭐
AuthN · rate-limit · transform · aggregate · BFF.
⑤ APP / SERVICE NODES ⭐
Stateless service · stateful service · background worker · cron · serverless function. Replicated visual (stack of boxes).
⑥ DATABASES · 6 FLAVORS ⭐
Relational · document · column (Cassandra) · key-value · graph · time-series. Distinct icons, distinct cost models.
⑦ CACHES ⭐
In-process · distributed (Redis/Memcached) · CDN · db query cache. Visual shows TTL + hit ratio.
⑧ QUEUES & STREAMS ⭐
Message queue (SQS · RabbitMQ) · pub-sub (Kafka · Redis) · event log (Kinesis) · job queue. Visual shows depth + lag.
⑨ OBJECT & FILE STORAGE
S3 / GCS / Azure Blob · file systems · block storage. Cost scales with GB + requests.
⑩ SEARCH & ANALYTICS
Elasticsearch · OpenSearch · Algolia · ClickHouse · Druid · Snowflake. Separate from OLTP databases.
⑪ OBSERVABILITY STACK
Metrics (Prom/Datadog) · logs · traces · APM · on-call. Often forgotten by juniors — explicit node teaches inclusion.
⑫ AUTH & IDENTITY
OAuth provider · SSO · IAM · secrets store · certificate authority.
⑬ 3RD-PARTY / EXTERNAL ⭐
Stripe · Twilio · Auth0 · OpenAI API · SendGrid. Cost-per-call. Reliability-not-yours. Teaches dependency management.
⑭ CONTAINERS · REGIONS · ZONES ⭐
Group-containers that hold other nodes. Drag-drop a node into a region/AZ. Visualizes placement.
⑮ REPLICATED / SHARDED BADGES ⭐
Any node can have a "×3 replica" or "×16 shards" badge. Visual shows replication factor without cluttering canvas.
⑯ EDGE TYPES · SYNC / ASYNC / BATCH ⭐
Solid = sync RPC. Dashed = async/fire-and-forget. Dotted = batch job. Edge style teaches call pattern.

Q49 · Diagram overlays & live views

Same architecture, many lenses. Click a button, canvas re-colors to reveal a different dimension. Multi-select.

① LATENCY HEAT MAP ⭐
Nodes colored by contribution to p99. Red = hottest. Click for breakdown.
② COST HEAT MAP ⭐
Nodes colored by monthly cost share. Instantly spot the expensive components.
③ BLAST RADIUS ⭐
Click any node → all downstream nodes that break if it dies, highlighted with severity.
④ REQUEST PATH TRACE ⭐
Trace one synthetic request through the system. Animated particle follows edges. Shows critical path.
⑤ DATA LOCALITY MAP
Overlays show where user data physically lives (region). Useful for GDPR / latency / sovereignty.
⑥ LAYERED VIEWS
Toggle: show only data layer · service layer · infrastructure layer. Progressive reveal.
⑦ ERROR RATE OVERLAY ⭐
Live during sim. Each node colored by current error %. Red nodes pulse.
⑧ ZOOM LEVELS
High-level (5 boxes), medium (services), low (per-instance). Semantic zoom, not just pixel zoom.
⑨ TIMELINE SCRUB
Bottom bar shows architecture at t=0, t=3mo, t=6mo, etc. Scrub to see how the design evolved. Great for growth-curve problems.
⑩ 3D ISOMETRIC VIEW
Optional view: physical rack visualization. Datacenters → racks → servers. Showpiece / pitch-deck mode.

Q50 · Smart canvas features — AI + pattern-aware canvas

The canvas is not just a drawing tool — it can recognize patterns, warn about anti-patterns, suggest improvements. Multi-select.

① PATTERN DETECTION ⭐
AI recognizes classic patterns in your design. Badges appear: "CQRS detected", "Leader-follower replication", "Saga pattern". Links to Learn content for each.
② ANTI-PATTERN WARNINGS ⭐
Detects problematic patterns: N+1 service call · cascade failure risk · hot-partition · missing observability · single-region-dependency · synchronous-chain-of-5+. Subtle red badge.
③ AI NODE SUGGESTIONS ⭐
"Given your current design, you're likely missing: ⬆ a cache between app and DB, ⬆ rate limiter at gateway." User can accept or dismiss.
④ COMPLEXITY SCORE
Each design gets a complexity number (services × edges × replication factor). Shown in corner. Teaches "is my design too elaborate?"
⑤ CONSTRAINT SOLVER ⭐
"Set p99 ≤ 50ms." AI suggests architecture changes to meet constraint. Shows the change as ghost-diff.
⑥ REVERSE ENGINEERING ⭐
Paste a system description ("Our service uses Postgres + Redis + Kafka + S3") → AI drafts a starter diagram. Turns text into canvas.
⑦ DRAG-IN REFERENCE COMPONENTS
Side library: "Netflix CDN stack", "Uber dispatch subsystem", "Stripe idempotency layer". Drag into your canvas → adapts to your design. Instant pattern reuse.
⑧ AUTO-LAYOUT PRESETS
One-click "tidy up": left-to-right · top-down · layered · circular. Dagre-based, serif-timed animation.

Q51 · Novel pedagogical formats

Beyond read-learn-drill. Formats competitors don't ship. Multi-select.

① BUILD-ALONG ⭐
"Follow along" sessions: senior architect builds a design step-by-step, narrating tradeoffs. User's canvas mirrors. Pause + replay. Like live coding but for systems.
② DEBUG-THIS ⭐
Given a broken design + symptoms ("latency spiked 6x at 2am") → diagnose. Teaches incident response. Rubric grades root-cause accuracy.
③ INHERIT-THIS-MESS ⭐
Given a poorly-designed system (real-world scenario: new employee Day 1). Refactor it. Teaches brownfield thinking — most engineering is NOT greenfield.
④ SCALE-THIS-UP ⭐
Working design at 10k users → modify for 100M. Shows what has to change at each order of magnitude. Teaches scaling inflection points.
⑤ CTO INTERVIEW
AI acts as CTO of a fictional startup. Presents a business problem ("we're losing $10k/day to latency"). You propose a design. They push back with business constraints. Teaches product-thinking alongside engineering.
⑥ GUIDED DERIVATION
Step-by-step textbook-style: "start with 1 server. now add scale. why does this break? let's add caching. why not perfect? and so on." Interactive, socratic.
⑦ COMPARE-TWO-COMPANIES
"Netflix vs Hulu · how do their video-delivery architectures differ?" Side-by-side canvases with annotated contrasts. Teaches "there's more than one right answer".
⑧ EVOLVING-DESIGN CAMPAIGN
Multi-hour narrative: "You join MockFlix Day 1. Scale is 100k users. In 6 game-months, you'll hit 100M. Survive each scaling wave." Each session is one chapter.

Q52 · Auto-generated documentation artifacts

Your design produces more than a diagram. What else gets auto-generated?

① ADR · ARCHITECTURE DECISION RECORDS ⭐
Each design choice auto-logged: "Decision: use Redis for session cache. Context: need <5ms lookup. Alternatives considered: Memcached, in-process. Consequences: ...". Industry standard, export as MD.
② RUNBOOK ⭐
"Given this design, when paged at 3am, here's your checklist." AI-generates incident response playbook per failure mode.
③ RFC-STYLE WRITE-UP ⭐
Full RFC doc: Goals · Non-goals · Design · Alternatives · Risks · Migration. Like Google/Meta internal RFCs. Export to Google Doc / Notion.
④ INFRA-AS-CODE STARTER
Export design as Terraform / CDK / Pulumi skeleton. Not deployable, but teaches "this architecture → this IaC file". Huge for junior eng career growth.
⑤ CAPACITY PLAN
"At 10M users, you need 40× app servers, 3× RDS primary, 8× cache nodes, ~$42k/mo." Spreadsheet-style projection doc.
⑥ POSTMORTEM TEMPLATE
Given a chaos-drill failure → auto-generates postmortem (timeline, root cause, impact, fix). Ready to paste into Confluence.
⑦ ONE-PAGER EXEC SUMMARY
"Design Netflix in 200 words for a non-engineer CEO." Teaches upward communication. Rubric grades clarity.
⑧ INTERVIEW CHEAT-SHEET
For a problem you've drilled: 1-page "what I'd say in the interview" crib. Scale math · 3 key tradeoffs · canonical diagram · failure mode. Your own voice.

Q53 · Visual / rendering innovations

Beyond standard ReactFlow look. How does the canvas feel?

① BLUEPRINT-PAPER MODE ⭐
Canvas background: subtle blueprint grid on cyan-white. Edges look hand-drafted. Strong "architect" signal. Unique visual signature.
② HAND-DRAWN MODE
Sketchy lines (rough.js style) during brainstorm phase. Signals "work in progress". Toggle to crisp mode when design is locked.
③ ANIMATED DATA FLOW ⭐
During simulation: particles/dots flow along edges. Visual "throughput". Faster = more requests. Dying edge = slowing flow.
④ NODE "BREATHING" ⭐
Nodes subtly pulse at their actual QPS rate. Feels alive. Faster pulse = higher load. Cinema-grade touch.
⑤ SERIF LABELS ⭐
Node labels in serif (IBM Plex Serif). Radically different from competitors' sans-heavy UIs. Signals "thoughtful" not "tech-bro".
⑥ AMBIENT SOUND (OPT-IN)
Low background hum during normal sim. Subtle bass pulse on chaos event. Silence when paused. Optional + off-by-default.
⑦ FAILURE CINEMATOGRAPHY ⭐
When a chaos event fires: screen vignettes dark, failed node glows red, 2s serif ribbon narrates. From Batch 5 = locked but noting here for completeness.
⑧ CONTEXT-AWARE NODE ICONS
Each node type has a distinctive, serif-styled icon (not generic). Cassandra node looks different from Postgres node. Intuitive at a glance.

Q54 · Immersion · narrative · worldbuilding

The highest-ambition lens. Does Architex feel like a place with characters and history? Or just a toolbox?

OPTION A · PURE TOOL
No narrative. Just content + canvas + simulation. Classical "textbook but interactive". Safe, understandable.
⭐ OPTION B · AMBIENT NARRATIVE
Architect's Studio as a place. Serif typography, earned color. Rooms for different activities. Cinematic chaos events. No characters or storyline — just atmosphere. This is already what we're building.
OPTION C · CHARACTER PROGRESSION
User has a title progression: Apprentice → Journeyman → Architect → Master. Each tier unlocks new interviewer personas / harder drills / esoteric content. Visible on profile.
OPTION D · FULL CAMPAIGN MODE
"The MockFlix Saga" — a 40-hour narrative where user is hired at a fictional company, grows from Eng1 to Staff, faces scaling crises, leads a rewrite. All content threaded through one story. Immersive; high authoring cost.
OPTION E · B + C
Atmosphere (locked) + title progression. Adds lightweight narrative without a campaign author. Safe middle.
OPTION F · B + D (DECADE SAGA)
Atmosphere + campaign. Most ambitious. Campaign is ONE long narrative optional layer — user can stick to discrete drills or follow the Saga.
