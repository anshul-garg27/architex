Q41 · Collaboration model

Is Architex a solo study app or a social tool? Different choices change the whole shape of the product.

OPTION A · SOLO ONLY
Every session is yours. No sharing, no peer review, no public anything. Calm, private, focused. But no viral loop.
⭐ OPTION B · SHAREABLE ARTIFACTS
Solo practice, but every completed drill/simulation produces a shareable link. Recipient sees the design + timer playback + rubric. Read-only. Subtle viral. Mentor-friendly.
architex.app/share/3f8a9b
→ "Daisy designed Netflix. Watch the 18-min replay + see rubric."
OPTION C · ASYNC PEER REVIEW
Submit your design to a peer queue. Another user reviews against rubric + leaves comments on specific nodes/edges. You review someone else's to unlock yours being reviewed.
OPTION D · LIVE PAIR SESSION
Two users on one canvas, real-time. One designs, one plays interviewer. Switch roles. WebRTC voice optional. Mimics mock-interview clubs.
OPTION E · B + C + D
Full social stack. Progressive: ship B first (easy), C next (queue), D last (real-time complexity). Most ambitious.

Q42 · Mobile experience

Canvas-first apps struggle on mobile. 60%+ of audience is mobile at least part of the time.

OPTION A · DESKTOP ONLY
"Designed for desktop. Mobile = read the learn content only." Honest. Polarizing.
⭐ OPTION B · MOBILE READ + REVIEW
Mobile fully supports: Learn mode, Review mode (FSRS cards), profile, share-link viewing. Build/Simulate/Drill are desktop-only (with a graceful "open on laptop" message). Flashcards during commutes — massive use case.
OPTION C · MOBILE CANVAS (READ-ONLY)
B + mobile can view (but not edit) Build/Drill canvases. Good for reviewing your own past designs or someone else's share link.
OPTION D · TABLET + STYLUS CANVAS
C + iPad/tablet gets a real canvas mode (finger / Apple Pencil). Simulates whiteboard interviews. Unique market position, more UX work.

Q43 · Content extensibility — can users/community add content?

Shipping 40 concepts + 30 problems + 10 real incidents + 73 chaos events is ambitious. At some point, we might let others contribute. When/how?

OPTION A · CLOSED CONTENT
Only Opus + the team author. Quality controlled. Slower to grow.
⭐ OPTION B · USER-AUTHORED DRILLS ⭐
Users can author their own problems: "Design my company's X". Private by default. Optionally submit to shared library. Moderator-curated. Grows catalog with community effort.
OPTION C · CHAOS PACKS
Third-party "chaos packs" — collections of curated chaos events around themes (e.g. "Fintech Pack: 20 chaos events relevant to payment systems"). Installable.
OPTION D · AI-ASSISTED AUTHORING
User gives rough idea ("design Airbnb but for pets"). Opus drafts full problem with scale numbers, constraints, canonical solution. User refines. AI accelerates content team.
OPTION E · B + C + D
User drills + chaos packs + AI-assist. Full extensibility. Ship B first (basic CRUD); C + D are future bets.

Q44 · Integrations — which matter?

Multi-select the integrations worth shipping in v1. Others deferred.

① EXPORT · IMAGE + MERMAID + JSON ⭐
Designs export to PNG, SVG, Mermaid code, JSON. Paste into any doc, resume, portfolio, Slack. Cheap and powerful.
② IMPORT FROM DRAWIO / EXCALIDRAW / LUCID
User drags their existing design into Architex → auto-parses to our schema → full simulation unlocked. Migrates existing designs into our sim.
③ GITHUB · SAVE + PUBLISH ⭐
Save designs as JSON/MDX to a user's Github repo. Public repos = discoverable portfolio. Integration with profile.
④ NOTION · SYNC NOTES
Learn mode bookmarks + drill rubrics push to a Notion page. User's personal SD wiki builds automatically. Notion-heavy users love this.
⑤ OBSIDIAN · VAULT EXPORT
Export entire knowledge graph as Markdown-linked Obsidian vault. For users who want offline/local-first. Small but passionate market.
⑥ LINKEDIN · PROFILE BADGE
Completed FAANG-Ready path → badge on LinkedIn. Recruiter-visible proof. Marketing flywheel.
⑦ CALENDAR · STUDY BLOCKS
Connect Google Calendar → Architex schedules 20-min daily review at your free slot. Streak preservation.
⑧ PUBLIC API
REST/GraphQL API over progress, designs, content. Power users + enterprise integrations. Big infra investment.

Q45 · Interview-week "Crunch Mode"

Real users have an onsite in 7 days. They need a compressed, intense prep path. Do we ship it?

OPTION A · NONE
Users self-pace. Works for months-long preparers; fails acute interviewers.
⭐ OPTION B · 7-DAY CRUNCH PATH ⭐
User inputs onsite date + target company. System generates a 7-day schedule: Day 1 foundation review · Day 2-3 drill core patterns · Day 4-5 mock full-stack loops · Day 6-7 light review + confidence drills. Honors calendar, nudges daily.
Your 7-day crunch — Meta E5
Mon · Consistent hashing + CAP + 1 drill (1h)
Tue · Pub-sub + sharding + 1 drill (1h)
Wed · Chaos drill: Design Twitter (1.5h)
Thu · Mock: Bar-raiser + Silent Watcher (2h)
Fri · Full-stack loop: Uber (1.5h)
Sat · Light: review cards + replay (30m)
Sun · Confidence drill + breathe
OPTION C · CRUNCH + COHORT
Same 7-day plan but synced with a cohort of peers. Shared progress, optional pair sessions. Gamified group accountability.
OPTION D · ADAPTIVE CRUNCH
User inputs start skill level (via diagnostic) + target date + hours available. AI generates a custom plan. Adjusts daily based on performance. Most personalized, most complex.

Q46 · Open mic — other ideas I should know?

One more chance to surface anything. Pick any that resonate — multi-select — or skip and we write the spec.

① "EXPLAIN LIKE I'M FIVE" MODE
Every concept has a parallel 300-word plain-English version. Toggle. For engineers learning during cross-domain transitions.
② AUDIO NARRATION
Every concept page has a 5-7 min audio version. Commute-friendly. TTS for draft, human-voiced for flagship pieces.
③ SEASONS & TOURNAMENTS
Quarterly "seasons" with themed drill sets ("Q2-2026: Fintech season"). Leaderboards within cohort. Ends → new season starts. Engagement loop.
④ PORTFOLIO MODE
Public profile page with your best designs, rubric scores, streaks, completed paths. Shareable URL. Recruiter-ready substitute for "system design on resume".
⑤ DIAGNOSTIC ENTRY QUIZ
First-session 15-min quiz across concepts + one quick drill. System auto-picks starting point. "You know caching well, weak on consensus — start here."
⑥ WEEKLY "DEEP-DIVE DIGEST"
Every Monday: email with 1 real-world incident write-up + 1 concept deep dive + 1 recommended drill. 5-min read. Builds habit + retention.
⑦ "BLAME AN ORG" POSTMORTEM DRILL
You get a real-world outage's symptoms (without attribution). Diagnose + fix in 20 min. Reveal: "this was actually Company X's 2022 outage". Teaches debugging + humility.
⑧ TEACHER / BOOTCAMP MODE
Educators create cohort accounts, assign problems, see student progress dashboards. Long-tail B2B revenue potential.
⑨ JOB-READY CERTIFICATION
Pass 5 Exam-mode drills with ≥4/5 rubric → earn "Architex Verified" badge. Companies can verify. Starts as vanity, becomes credential.
⑩ "TALK-ALOUD" MIC + TRANSCRIPT
During any mode, optionally record your voice + auto-transcribe. Replay hears your reasoning. Critically: the rubric grades what you SAID, not just what you drew. Unique in market.
⑪ DARK / LIGHT / EARTH-TONE THEMES
Dark (default) · Light (for OS light mode) · "Earth" (warm parchment for long reads). Accessibility + mood.
⑫ "REFERENCE ARCHITECTURE" LIBRARY
30+ full reference architectures for real systems (published with sources): Netflix actual 2018 CDN, Uber 2022 matching, Stripe 2023 ledger. Explorable in canvas, with author commentary.
