Batch 7 of 10 · Cost & scaling as learning surfaces. Turning the dormant cost model into design pressure. Teaching engineers to feel the dollar signs.

The premise · Cost is the invisible force that bends every real design

Interview candidates hand-wave infrastructure. Real engineers don't. At $400k / year, a senior is expected to know: "3 Redis clusters across regions costs ~$8k/mo, is that worth it vs. eventual consistency?" The simulation engine already tracks cost per node. This batch turns that number from display to teacher.

⚖️
THE INVISIBLE-FORCE PRINCIPLE
"Perfection isn't when there's nothing left to add — it's when there's nothing left you can afford."
Every design decision has a price. Caching: cheaper reads, more infra. Multi-region: better DR, 2-3x cost. Microservices: nicer boundaries, 10x operational overhead. Textbooks skip this. Real architects live it. Our simulation must make the price visible, provocative, and sometimes constraining.

Q23 · When is cost visible during design?

Cost is either always-on (background pressure) or summoned (reflective moment). The choice shapes the whole experience.

OPTION A · HIDDEN BY DEFAULT
Cost invisible unless user clicks "Cost" tab. Matches Build mode minimalism. But: beginners never think about it.
OPTION B · CORNER METER
Always-visible cost meter in bottom-right corner. Updates as user drags nodes on canvas. Green under budget, amber approaching, red over.
💰 $6,840 / mo • 68% of budget
⭐ OPTION C · PER-NODE + METER + DIFF
Full transparency. Each node shows its monthly cost on hover. Corner meter shows total. When you add/remove a node: "+$740/mo" floating annotation fades in. Teaches the unit economics.
⬤ postgres-primary $420/mo (db.m5.xlarge)
⬤ redis-cache $180/mo (cache.m5.large)
⬤ cdn-tier $240/mo (CloudFront · 5TB egress)
+ new node: +$740/mo
OPTION D · MODE-DEPENDENT
Build mode: hidden by default. Drill mode: corner meter (stakes present). Simulate mode: per-node + meter + diff (teaching moment). Cost appears where it matters most.

Q24 · Cost model scope — which providers do we ship?

AWS pricing differs from GCP differs from Azure. Cost modeling is real engineering work.

① AWS ⭐
70% of prep candidates deploy here
EC2, RDS, ElastiCache, S3, CloudFront, Lambda, DynamoDB, SQS, MSK. ~30 services priced. Industry default.
② GCP ⭐
for Google-aspirant candidates
GCE, Cloud SQL, Memorystore, GCS, Cloud CDN, Cloud Run, Firestore, Pub/Sub, BigQuery. ~25 services.
③ AZURE
enterprise-prep bucket
VMs, SQL DB, Cosmos DB, Cache for Redis, Blob Storage, Front Door. For candidates targeting MSFT / enterprise.
④ PROVIDER-AGNOSTIC (ABSTRACT UNITS) ⭐
the pedagogical default
"small instance $50/mo", "large instance $400/mo", "cache tier $200/mo". Rounded, memorable. Teaches cost-thinking without provider noise. Toggle to "real AWS prices" when user is ready.
⑤ CROSS-PROVIDER COMPARE
power-user feature
Same design priced side-by-side across AWS/GCP/Azure. Shows DynamoDB vs. Firestore vs. Cosmos DB cost lines. Reveals where egress & storage pricing bite.
⑥ SELF-HOSTED / BARE METAL
reality-check option
"This same design on colo racks at Hetzner costs ~1/5. Why does cloud cost 5x?" Teaches the cloud premium. Strong signal of senior thinking.

Q25 · Scaling scenarios — how does scale enter the design?

A design for 1k users is not a design for 100M users. The simulation lets us move along that curve.

OPTION A · FIXED TARGET
Each problem specifies one scale. "Design Netflix for 200M DAU." User designs to that. Clean, pedagogical.
⭐ OPTION B · SCALE SLIDER
User drags slider: 10k → 1M → 10M → 100M → 1B DAU. Cost updates. Latency updates. Whole architecture may need to change. Teaches "what breaks when you grow 10x?" — the core SD question.
▼ SCALE: 1M DAU
├───●────────────────── 10k · 1M · 10M · 100M · 1B
cost: $8,400/mo → p99: 180ms → shards: 1
OPTION C · GROWTH CURVE
"You start with 1k users. In 12 months you're at 10M. Design for that journey." Timeline scrub. Shows what breaks and when. Teaches capacity planning.
OPTION D · B + C
Slider for static "point-in-time" view. Curve for trajectory view. Toggle. Covers both mental models.

Q26 · Cost-pressure mechanics — ship these activities?

Cost is most instructive when it constrains. Multi-select which activities we ship.

① BUDGET-CONSTRAINED PROBLEMS ⭐
Each problem has a monthly budget ("Design Netflix on $50k/mo infra"). Tight enough to force tradeoffs. Violating budget = rubric penalty. Teaches "what do we cut?"
② COST-OPTIMIZATION DRILL ⭐
Given a working but expensive design, reduce cost by X% without breaking SLOs. Like a refactor drill. Teaches reserved instances, spot, right-sizing, caching at right layer.
③ "WHAT COULD WE CUT?" MODE ⭐
AI analyzes current design, suggests 3 cost-cut options with SLO impact estimates. "Drop multi-region to single-region: −42% cost, +15% chance of regional outage." Every tradeoff explicit.
④ COST-PER-USER EXPOSED ⭐
"Your design costs $0.012 per user per month. Netflix spends ~$0.40. YouTube ~$0.08." Normalizes scale. Makes "is this cheap or expensive?" answerable.
⑤ FORECAST COST CURVE
Given growth rate, plot cost over 24 months. Shows nonlinear jumps ("shard split needed at 10M users = +$8k/mo"). Teaches capacity budget planning.
⑥ COST VS p99 TRADEOFF CURVE
Plots: each architectural option at a point on (cost, p99) plane. User sees the Pareto frontier. Teaches "at what price do we get what latency?" — the core senior-eng skill.
⑦ "CLOUD BILL SHOCK" SCENARIO
Real narratives from companies that blew their AWS bill. "One engineer enabled CloudTrail on all regions — $200k/mo." Teaches cost-observability and guardrails.

Q27 · Hidden & operational costs — which to surface?

The cost model is easy to do wrong. These are the non-obvious cost lines that separate juniors from seniors. Multi-select.

① DATA EGRESS ⭐
AWS charges $0.02-0.09 per GB out. Cross-region transfer is worse. 1 PB out = $40k-90k. Easily surprises junior engineers.
② OBSERVABILITY TAX ⭐
Datadog/Splunk/Sentry/APM often 10-30% of compute cost. High-cardinality metrics can 10x bills. Real number most engineers underestimate.
③ ON-CALL LOAD
Not a cloud bill — but real. "This design needs 3 on-call engineers × $300k ea = $900k/yr hidden cost." Teaches operational honesty.
④ VENDOR LOCK-IN COST ⭐
DynamoDB cheaper than running Cassandra — but what's the cost of migrating off? Proxy metric: how many vendor-specific APIs does your design touch?
⑤ CROSS-REGION REPLICATION
Multi-region DR sounds great — costs 2-3x (compute + egress + sync). Shows up clearly when slider hits multi-region.
⑥ IDLE-CAPACITY WASTE
Over-provisioning. "You've got 3 app servers at 12% CPU." Teaches right-sizing + autoscale tuning. Most common real-world cost leak.
⑦ RETRY AMPLIFICATION COST
Bad retry logic → 3-5x actual requests → 3-5x costs. Hidden. Shows during chaos simulation. Teaches backoff + circuit breakers as cost mechanisms.
⑧ COMPLIANCE / AUDIT
Adds encryption + logging + region segregation. SOC2 = +15-25% infra cost. HIPAA / PCI more. Most textbooks never teach this.

Q28 · Real-company cost references — how much do we expose?

Actual numbers from Netflix / Reddit / Twitter build intuition. But some numbers are proprietary / stale.

OPTION A · ABSTRACT ONLY
No named-company numbers. "Large streaming service: ~$0.10-0.50 per user/month." Safe, stale-resistant. But loses teaching punch.
⭐ OPTION B · NAMED BANDS (ROUNDED)
Disclosed public numbers, rounded, with year stamp + source. "Netflix AWS spend ≈ $400M/yr (2022 earnings call)." Memorable and sourced.
Netflix: ~$400M / yr AWS · ~$0.20 / user-month (2022)
Reddit: ~$100M / yr infra · ~$0.05 / user-month (2023 S-1)
Dropbox: saved $75M / yr moving to Magic Pocket (2016)
Snapchat: paid $3B over 5 yrs for Google Cloud (2017 deal)
Sources: earnings reports, S-1 filings, engineering blogs.
OPTION C · INFRA PER USER GRAPHS
Visualization: log-log plot of user-base vs. infra-cost for 15 companies. User's design plotted on same graph. Visceral "am I in the ballpark?" check.
OPTION D · B + C
Named bands in tooltips + graph for visualization. Both channels. More work but higher pedagogical density.
