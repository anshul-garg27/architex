The premise · A failure is just data until it becomes a story

The engine has 73 chaos events already wired. Right now they're visual effects with state mutations — a red pulse, a broken edge, a metric spike. Engineers remember stories, not metrics. "Facebook Oct 2021" is a story. "Cache eviction at t=0:47" is a log line. This batch decides how to promote every chaos event from log line to story.

📖
THE WAR-STORY PRINCIPLE
"Senior engineers don't remember concepts — they remember the 3 AM page that taught them the concept."
A chaos event should enter the user's memory as a story: a scene, a character, a stakes-moment, an aftermath. "The time the cache evicted during the demo" beats "cache eviction can cause latency spikes" in every cognitive science metric: recall, transfer, retention. Each of our 73 events is a potential story. This batch decides the shape of those stories.

Q11 · How do we group 73 chaos events for the user?

Users need to browse, pick, search. 73 is a lot. Taxonomy decides the first screen of chaos mode.

⭐ OPTION A · BY FAILURE DOMAIN (7)
Group by where the failure originates. Matches engineers' mental model.
🔥 Infrastructure (14) · region down, disk full, CPU spike, OOM
🗄 Data layer (11) · DB primary down, replica lag, shard imbalance
🌐 Network (10) · latency spike, packet loss, DNS flap
🌊 Cascade (9) · thundering herd, retry amplification, cache storm
🔌 External dep (8) · 3rd-party API, CDN, payment gateway
👤 Human (10) · bad deploy, config drift, deleted index
📈 Load (11) · viral spike, bot attack, hotspot
OPTION B · BY SEVERITY
Group by blast radius. Good for difficulty sorting but weak for learning — the same "minor" event (slow disk) can cascade differently in different systems.
⬤ Catastrophic (12) · total outage
⬤ Severe (25) · multi-service degradation
⬤ Moderate (23) · single-service
⬤ Nuisance (13) · recoverable glitch
OPTION C · BY REAL-WORLD ANALOG
Name each event after a famous incident it resembles. "The Facebook 2021", "The Cloudflare Regex", "The us-east-1 Cascade". Memorable but locks you into naming.
OPTION D · A + B DUAL FILTER
Primary browse by domain (A). Severity shown as colored dot per event. User can filter either way. Best of both but requires richer UI.

Q12 · What does one chaos event feel like when it fires?

The instant an event triggers — the ribbon, the tone, the voice. This is the 5-second hook that turns a metric into a memory.

OPTION A · INCIDENT REPORT
Terse, clinical. Like a production runbook.
[INC-047] PRIMARY DB UNREACHABLE
Severity: P1 · Region: us-east-1
Started: 03:47:22 · Detected: 03:47:34
Blast radius: writes, 73% of reads
Pros: looks professional. Cons: dry, forgettable.
⭐ OPTION B · CINEMATIC NARRATOR
Third-person, present-tense, stakes implied.
3:47 AM Pacific. Black Friday. The primary database in us-east-1 just stopped answering. Your read replicas are still alive — for now. In 90 seconds, they'll start falling over from write backpressure. What do you do?
Pros: memorable, cinematic, tested in industry training. Cons: authoring time per event.
OPTION C · ENGINEER DIARY
First-person. The user IS the on-call.
"PagerDuty woke me up. I fumble for my laptop. Dashboards are red. DB primary — dead. Why now? Why this night? I'm opening the runbook but there's no time — writes are already queueing..."
Pros: visceral. Cons: can feel performative.
OPTION D · DIALOG BETWEEN ENGINEERS
Two voices. Teaches through conflict.
Sarah: "why is p99 at 3 seconds?"
Jake: "retries. Everyone's retrying the dead DB."
Sarah: "...so we're DDoSing ourselves."
Jake: "yup. welcome to thundering herd."
Pros: teaches by mimicking real postmortems. Cons: 2x authoring.

Q14 · How does the user control chaos?

In Chaos Drill mode, the user is on-call. In other modes, they're more like a director. Multi-select the control modes to ship.

① SCENARIO SCRIPT ⭐
Pre-written sequence. "Black Friday Netflix" runs events 1→2→3 at predictable times. Used by Chaos Drill + real-incident replays. Must-ship.
② CHAOS DICE ⭐
Weighted random. Every 45-90s a random event from the active domain fires. Unpredictable, replayable with seed. Used by Stress Test.
③ MANUAL INJECTION
Chaos panel lists all 73 events. User clicks to fire specific one. For Build/Validate mode — test specific hypotheses. "What happens if my cache dies right now?"
④ CHAOS BUDGET ⭐
Gamified: "you have 3 chaos tokens in 5 minutes. Plan wisely." User picks when to fire which event, watches design respond. Teaches thinking in failures.
⑤ AUTO-ESCALATION
Engine detects cascade triggers (cache evicted → DB hot). Auto-fires secondary events from the cascade graph. Simulates realism.
⑥ RED-TEAM MODE
AI (Sonnet) picks the worst possible chaos given your design's weakness. It reads your architecture, finds the fragile node, breaks it first. Brutal teacher.

Q15 · Visual + audio treatment when chaos fires

The 5-second hook. Too loud = anxiety. Too quiet = ignored. Pick the vibe.

OPTION A · MINIMAL DESIGN
Small toast appears top-right. Affected node glows amber for 2s. No audio. Serif voice.
Calm, respectful. But feels like nothing happened.
⭐ OPTION B · CINEMATIC PULSE
Full-width narrative ribbon slides in (2s, auto-dismiss 8s). Affected nodes pulse red with ring-of-influence. Screen edge briefly vignettes red. Optional subtle bass thump. Voiceover OFF by default (accessibility + library).
Urgent but not anxiety-inducing. Every animation serif-timed (550ms, not 200ms).
OPTION C · WAR ROOM
TV-news aesthetic. Red banner, alert sound, Pagerduty-ish beep. Good for demos and engagement. But sustained anxiety for long sessions.
OPTION D · PERSONA-SCALED
Rookie gets option A (gentle). Journeyman gets B. Architect persona gets optional C (full war room with sound). Respects the user's tolerance.

Q16 · What happens AFTER a chaos event fires?

The afterglow. The moment between "that just happened" and "here's what I learned".

OPTION A · NOTHING SPECIAL
Metrics settle, sim continues. Everything gathered into post-mortem at end of run.
⭐ OPTION B · MARGIN CARD
Small card appears in right margin with one sentence lesson + "learn more" link. Stacks with previous events. User reads in their own time. Creates a visible "storyboard" of what happened during the run.
▸ cache eviction storm · t=0:47
"TTL sync across all keys caused simultaneous expiry → DB hit. Fix: jittered TTL."
→ Learn: Caching Strategies
OPTION C · PAUSE & REFLECT
Each event auto-pauses sim for 8s with modal: "pause to absorb this?" User can continue or drill in. Interrupts flow but guarantees learning moment.
OPTION D · ACHIEVEMENT CHIP
First time you survive each chaos event → collectible "achievement" chip. "First thundering herd survived." Gamified. Stored in profile.
