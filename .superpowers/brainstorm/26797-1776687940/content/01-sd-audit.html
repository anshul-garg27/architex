What I found inspecting System Design today

Live browser + code inspection complete. Verdict: powerful infrastructure, zero instructional scaffolding.

✅ WHAT EXISTS · RICH
Canvas infrastructure (excellent): React Flow + 75+ node types across Compute/Messaging/Data/Networking/FinTech/AI/Observability · A*-style edges · minimap · 53 built-in templates · Cmd+K palette
Simulation engine (enormous, mostly dormant): src/lib/simulation/* has 34 files, ~2,500 LOC — orchestrator, chaos-engine (73 events), cascade, cost-model (~75 AWS components), time-travel, queuing model, SLA calculator, capacity planner, narrative engine
Bottom panel tabs wired: Metrics · Timeline · SLA · Latency · Code · Console · Chaos · Capacity · plus more. Great UI surface area.
Visual polish: Glassmorphic dark theme, motion/react, particle layers already in codebase, smooth interactions
❌ WHAT'S MISSING · CATASTROPHIC
ZERO learning layer. LLD has 36 patterns as study content. System Design = just a blank canvas. No concepts to learn, no problems to solve with guidance, no lessons at all.
NO PROBLEMS CATALOG. Industry standard: "Design Netflix · Design Uber · Design Twitter · Design WhatsApp" etc. Architex has zero. LeetCode has 50+. ByteByteGo has 200+.
Simulation doesn't actually demonstrate learning. Play button exists. Start a sim → metrics MIGHT show. But there's no lesson tied to what you're observing. A learner sees numbers without context.
Chaos events are data-driven in code, not in UI. 73 chaos event types defined. No visible "inject Netflix-style failure here" UX. Just a Chaos tab showing what?
No scoring · no drills · no retention loop. No way to "practice system design interview" with feedback. No FSRS review. No measurable skill progression.

  properties: <PropertiesPanel />,
  bottomPanel: <BottomPanel />

Four primitives stitched together. That's it.

💡 THE INSIGHT
System Design was built as a professional drafting tool — think Figma for backend architecture. LLD was built as an educational tool — think Brilliant for design patterns. They started with different DNA.

Our task: graft LLD's educational DNA onto System Design's powerful simulation muscle. Then add system-design-specific pedagogy (scalability drills, chaos stories, cost thinking) that LLD can't offer.

Result: the only place in the world where you can design Netflix, simulate production traffic, break it, and get graded on it.
