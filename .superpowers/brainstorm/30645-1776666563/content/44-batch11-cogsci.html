CS1

Confidence-weighted checkpoints · metacognition calibration

Before answering, user rates confidence 1-5. Score = accuracy × confidence. Overconfidence + wrong = bigger penalty. Humble + right = bonus. Teaches learners to know what they don't know — critical interview skill.

CS3

Elaborative interrogation · "why?" prompts

After each lesson, 3 "why" prompts force the learner to generate explanations: "Why is Observer better than callbacks here?" · "Why would this fail at scale?" · "Why did the author choose composition over inheritance?". Free-text answer. AI grades for depth (not correctness). Generating explanations = 2x retention.

CS4

Productive failure · design BEFORE being taught

Manu Kapur research (cited in your README): learners who struggle at a problem before being taught perform 2-3x better. New format: pre-lesson design challenge. Given requirements, design it yourself (20 min). Then the lesson shows the "canonical" pattern. Contrast = deeper learning.

Step 1 · Try it first (20 min)
"Multiple subscribers need to react when stock price changes. Design it."
Step 2 · Reveal the canonical pattern
"This is the Observer pattern. Compare yours →"
Step 3 · What you almost got · what you missed
"You nailed the broadcast idea. You missed: decoupling the Subject from concrete observer classes. Look closer..."
Toggle: "Classic lesson" vs "Productive Failure lesson" — let user pick. 2x retention for motivated learners.

CS7

Teach-back mode · explain to an imaginary intern

Feynman technique. User types or records a 2-min explanation as if teaching a junior engineer. AI plays the intern — asks follow-ups. ("Wait, why private? What if it's public?"). Generates 3-5 probing questions. Answering them locks the concept.

🎓
"Singleton means one instance. We use a private constructor so nobody can call new Logger(). And a static getInstance() that makes the first one if needed."
🧑‍💼
Intern: "Hmm, what happens in a multi-threaded app if two threads call getInstance() at the same time? Don't we end up with two instances?"
Can't answer = you don't really understand it. Feynman test.
CS8

Anti-pattern hunt · find the bug in someone's design

Given a UML canvas with a subtly broken design (or anti-pattern), find the issue in 5 min. Examples: "Observer registered but never unregistered → memory leak" · "Singleton used as global state → hidden coupling" · "Inheritance where composition belongs". Trains code review skills.

🐛 SPOT THE BUG · 5:00 remaining
class StockPrice {
  subscribe(obs) { this.subs.push(obs); }
  notify() { this.subs.forEach(o => o.update()); }
  // missing: unsubscribe
}
Your answer: "Memory leak — no unsubscribe method..."
CS9

Design debate · defend a pattern choice against AI

Pick a pattern. AI takes the opposite side. "Why Observer and not Publisher-Subscriber?" · "Why Factory Method here? Isn't Builder better?". 3 rounds of argument. AI grades: Were your reasons specific? Did you concede valid counterpoints? The actual interview skill.
