Q2 · In what order should the 36 patterns be learned?

Beginner opens Learn mode. Sidebar shows 36 patterns. What do they see first — a curated path, a free menu, or a locked sequence? This shapes onboarding for personas A & B especially.

A

Pure linear · locked sequence

Pattern 2 is locked until Pattern 1 is complete. Classic "course" model. Duolingo-style gates.

✓ Singleton      (done)
▶ Factory Method (current)
🔒 Abstract Factory (locked)
🔒 Builder
🔒 …33 more
Pros: Zero decision paralysis. Perfect for persona A. Clear progress signal.
Cons: Frustrates persona C (crammer). Can't review old patterns unless completed ones are revisitable. Feels infantilizing to pros.
B

Prerequisite DAG · earn-your-way-in

Patterns have explicit prerequisites. E.g., Abstract Factory depends on Factory Method. Strategy depends on basic Interface understanding. Graph-based, not linear.

✓ Singleton
→
▶ Factory Method
↓
🔒 Abstract Factory (needs Factory Method)
▶ Observer
→
🔒 Mediator (needs Observer)
Pros: Pedagogically honest — real dependencies enforced. Can work on multiple branches in parallel.
Cons: Authoring the DAG is a ton of work. Still gates pros. Complex to visualize.
C

Free-choice · no gates · like LeetCode

All 36 patterns visible and clickable from day one. No locks. Sort by difficulty / category / frequency. User picks what they want.

Sort: [Difficulty ▼] [Category] [Frequency]
● Singleton      Easy
● Factory Method Easy
● Observer      Medium
● Visitor       Hard
…all 36 clickable
Pros: Pros love it. Zero friction. Respects user intelligence.
Cons: Persona A is paralyzed — "where do I even start?" Dumps all 36 on them.
D
⭐ MY RECOMMENDATION

Hybrid · recommended path (DAG) + "free explore" escape hatch

Default view shows a curated learning path — 36 patterns ordered in a DAG, highlighted node = "start here", next node = "suggested next". But nothing is locked. A toggle [Guided · All patterns] swaps to flat list for pros.
