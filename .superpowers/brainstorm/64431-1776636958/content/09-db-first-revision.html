DB-first persistence — the revised strategy

DB is source of truth. localStorage becomes an offline cache. Mutations write through both. On mount, hydrate cache fast then reconcile from DB.

🌍
CROSS-DEVICE

📊
ANALYTICS-READY
Every mode switch, drill, completion becomes queryable. Product insights by default.
🔒
DURABLE
Clear browser cache? Everything survives. Switch browsers? State follows.

One new DB table: lld_drill_attempts

Serves both active drills AND completion history. State machine: active → submitted | abandoned.

// src/db/schema/lld-drill-attempts.ts (new)

export const lldDrillAttempts = pgTable("lld_drill_attempts", {
  id: uuid("id").primaryKey().defaultRandom(),
  userId: uuid("user_id").notNull().references(() => users.id, { onDelete: "cascade" }),
  problemId: varchar("problem_id", { length: 100 }).notNull(),

  // Lifecycle timestamps
  startedAt: timestamp("started_at", { withTimezone: true }).notNull().defaultNow(),
  pausedAt: timestamp("paused_at", { withTimezone: true }),  // null = running or done
  lastActivityAt: timestamp("last_activity_at", { withTimezone: true }).notNull().defaultNow(),
  submittedAt: timestamp("submitted_at", { withTimezone: true }),  // set on submit
  abandonedAt: timestamp("abandoned_at", { withTimezone: true }),  // set on give-up or auto-abandon

  // Timer accumulator
  elapsedBeforePauseMs: integer("elapsed_before_pause_ms").notNull().default(0),
  durationLimitMs: integer("duration_limit_ms").notNull(),

  // Canvas snapshot + interaction log
  canvasState: jsonb("canvas_state"),  // nodes + edges as user builds
  hintsUsed: jsonb("hints_used").notNull().default([]),  // [{tier, usedAt}]

  // Grading result (filled after submit)
  gradeScore: real("grade_score"),  // 0-100
  gradeBreakdown: jsonb("grade_breakdown"),  // {classes, relationships, patternUsage, completeness}
}, (t) => [
  // Enforce "only one active drill per user"
  uniqueIndex("one_active_drill_per_user")
    .on(t.userId)
    .where(sql`${t.submittedAt} IS NULL AND ${t.abandonedAt} IS NULL`),
  index("drill_history_idx").on(t.userId, t.submittedAt),
]);
Partial unique index magic: A user can have many COMPLETED drills (submitted or abandoned) but only ONE active at a time. This is enforced at the DB level — no race conditions possible. Try to start a second drill while one is active → DB rejects with constraint violation → UI catches it, offers "Abandon current drill and start new one?"

Handling stale drills (the key offline edge case)

What if user closes tab mid-drill and comes back 2 days later? DB has their drill with pausedAt set.

HEARTBEAT (every 10s while active)
PATCH /api/lld/drill-attempts/:id with { lastActivityAt: now }. Keeps a liveness signal server-side.
ON USER RETURN → GET /api/lld/drill-attempts/active
Server checks: if lastActivityAt < now - 30 minutes → auto-abandon it (abandonedAt = now) and return fresh state. UI shows: "Your last drill was abandoned. Restart?"
IF RECENTLY ACTIVE (< 30 min ago)
Show resume prompt: "You have an active drill (Parking Lot · 12:34 remaining). Resume / Submit now / Give up?"

New API routes (6 endpoints)

GET   /api/user-preferences
→ { lld: { mode, welcomeBannerDismissed, scratchCanvas, ... } }

PATCH /api/user-preferences/lld
→ Partial update of lld preferences (debounced write-through)

GET   /api/lld/drill-attempts/active
→ Current active drill OR null. Auto-abandons stale (>30min idle).

POST  /api/lld/drill-attempts
→ Start a new drill. 409 if one's already active.

PATCH /api/lld/drill-attempts/:id
→ Pause / resume / heartbeat / submit / abandon + canvas state snapshot

GET   /api/lld/drill-attempts?status=completed
→ History for review / stats dashboard

Write-through sync hook

Single reusable pattern: mutate locally for instant UI, then sync to DB in background.

// hooks/useLLDPreferencesSync.ts

export function useLLDPreferencesSync() {
  const { lldMode, lldWelcomeBannerDismissed } = useUIStore();
  const mutation = useMutation({
    mutationFn: (patch) => fetch("/api/user-preferences/lld", {
      method: "PATCH", body: JSON.stringify(patch),
    }),
    // networkMode: "offlineFirst" → queues if offline
  });

  // Debounced sync: 1s after last change
  const debouncedSync = useMemo(() =>
    debounce((patch) => mutation.mutate(patch), 1000), []);

  useEffect(() => {
    debouncedSync({ mode: lldMode, welcomeBannerDismissed });
  }, [lldMode, lldWelcomeBannerDismissed]);
}

The logged-out edge case

Clerk is optional. What about anonymous users?

🔓 ANONYMOUS
localStorage only. Full UX works except cross-device sync and SRS-tracked review queue. Mode preference, canvas, learn progress all local. Drill works but no history saved.
🔑 ON FIRST LOGIN
Migration: read current localStorage state → POST to /api/user-preferences/lld/migrate with anonymous state. Server merges into user row. Subsequent loads = DB-first.
✅ SUMMARY OF CHANGES VS PREVIOUS SECTION
  • 1 new DB table: lld_drill_attempts (serves active + history)
  • Reuse existing tables: userPreferences (extend JSONB), progress (extend JSONB metadata), activityEvents (log mode switches + checkpoint answers)
  • 6 new API routes for preferences + drill lifecycle
  • Write-through pattern: every store mutation triggers debounced DB sync via TanStack Query (already configured offlineFirst)
  • localStorage demoted from source-of-truth to fast cache
  • Anonymous user support via local-only mode + migration on login
