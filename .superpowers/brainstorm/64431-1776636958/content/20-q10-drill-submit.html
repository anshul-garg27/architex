Q10 · User clicks Submit in Drill. What happens next?

This is the most emotionally loaded 3 seconds in the entire product. A good reveal creates a dopamine loop worth grinding for. A bad one feels like a test. Get this right and users come back. Get it wrong and Drill dies.

🎭 TWO SCHOOLS OF SUBMISSION UX
LeetCode school

Raw results. "Accepted" or "Wrong Answer". Test cases listed. Move on. No celebration, no hand-holding. Respects pros.

Duolingo school

Celebration every time. XP spray, level-up sounds, dopamine drip. Builds habit at the cost of feeling infantilizing when you're experienced.

A

Instant dump · LeetCode-style

Click Submit → score + breakdown appear instantly at top of panel. No animation, no transition. "You scored 73/100. Classes: 32/40. Relationships: 21/30. ..."

Pros: Fast. Respects time. Pros don't feel patronized.
Cons: Flat — no emotional payoff for success. Failure feels brutal with no coaching. Habit loop weaker.
B
⭐ MY RECOMMENDATION

Tiered reveal · celebration scales to earned quality

3-second choreography: grading spinner → score emerges → breakdown unfolds → action choices. Visual treatment varies by tier — confetti only when earned (≥90%), specific coaching when failing. Never brutal, never infantilizing.

C

Interactive walkthrough · user expands each rubric category

After submit, show 4 category cards (Classes, Relationships, Pattern Usage, Completeness) as tabs. User clicks each to see specific feedback with diff.

Pros: Controlled pacing. Deep-dive per category.
Cons: More clicks. No celebration moment. Feels like a tax form.
D

Side-by-side compare with reference solution

Submit → screen splits · user's canvas on left, reference solution on right · click differences to see explanations.

Pros: Visual feedback. See exactly what's different.
Cons: Reveals reference too easily = less incentive to retry. Also assumes 1 canonical solution (LLD problems have many valid designs). Dishonest for designs that are different but correct.
💡 WHY B · EARNED CELEBRATION, COACHED FAILURE
The tier philosophy: 90+ earns confetti. 70-89 earns respect (no celebration, specific tune-up). 50-69 earns honest partial credit + path forward. <50 earns compassion + a strategic next step (not "try again").

Dopamine asymmetry is a feature. If EVERY submission triggered confetti, confetti stops meaning anything. If NO submission ever did, the 95% moment feels like the 40% moment. Tiered treatment makes each outcome feel distinct.

Failure-coaching is where Architex differentiates. LeetCode says "Wrong Answer". Brilliant says "Try again!" Duolingo says "Heart lost!". None of them tell you what specifically was wrong and what to do about it. Your fuzzy grader + pattern detection already has this data — use it.

Tier → next-action mapping is where retention lives. A 40% score that ends with "next problem →" is brutal. A 40% score that ends with "Open Learn mode for State pattern — 5 min" is a path. Path beats grade.
