Batch 6 · Layout & canvas architecture deep-dive. From my browser inspection of the live LLD module, here's where the current layout has room to grow in sophistication.

How the canvas shell itself could be smarter

Multi-select. Layout ideas that the live LLD UI either lacks or could level up.

L1

Resizable panels everywhere · drag-to-resize sidebar/canvas/properties

Currently panel widths are fixed. Engineers want control — give them drag handles at panel borders. Preferences persist per-user. Mini-map auto-positions. Your stack already has react-resizable-panels.

~4h. Library already in deps.
L2

Collapsible panels · one-click hide sidebar / properties for focus

Hit the chevron → sidebar slides away, canvas expands to claim space. Same for properties panel. Keybindings: ⌘[ collapse sidebar, ⌘] collapse properties. Muscle-memory from VS Code.

~3h. Chevron UI + state persistence.
L3

Pattern tabs · browser-style tabs for open patterns/problems

Open Singleton. Click Observer — opens in a new tab. Switch between tabs without losing scroll/state. ⌘W closes. Pros familiar pattern from VS Code / Figma.

~10h. Each tab persists canvas state.
L4

Split-view · compare two patterns side-by-side in one window

Hit "compare" icon → canvas splits vertically. Left: Observer. Right: Mediator. Both scrollable/editable independently. See structural differences at a glance. Teaching feature + crammer tool.

~15h. Two canvas instances, shared chrome.
L5

Distraction-free "zen mode" · everything fades but the canvas

Keybind ⌘.. Sidebar, properties, header, status bar all slide away. Canvas fills the viewport. Useful when designing in flow state. Another keybind to restore.

~3h.
L6

Canvas zoom semantics · pattern-context-aware fit-to-view

Hitting "Fit" in current UI fits the whole diagram. Propose richer zoom: Fit selection, Fit group, Fit highlighted, Reset to 100%. Each as discrete menu option. Power users want control.

~4h. React Flow has zoom APIs.
L7

Snap-to-grid toggle · visible grid on hold-Shift

Normal mode: free positioning. Hold Shift while dragging: snap to invisible 8px grid + temporarily show grid lines. Release: grid vanishes. OCD-friendly, optional.

~2h.
L8

Alignment guides · live guides appear when dragging classes

Dragging a class near another's edge → pink dashed alignment guides appear when edges would match. Release = perfect alignment. Figma-style. Tiny detail, enormous quality signal.

Your codebase already has AlignmentGuides + DistanceIndicators components. Wire them into LLDCanvas. ~3h.
L9

Right-click context menus · reveal advanced actions

Right-click a class → menu: Edit, Duplicate, Delete, Add method, Add attribute, Convert to interface, Extract parent, Copy name, Copy as code, Copy as Mermaid, Find usages in lesson. Packs 10 actions into 1 shortcut. Industry standard.

CanvasContextMenu exists — extend with LLD-specific actions. ~6h.
L10

Status bar polish · live context + micro-stats

Current status bar shows "0 nodes / 0 edges". Propose richer context: "Factory Method · 6 classes · 4 relationships · 92% completion · last saved 2s ago". Live, compact, informative. Click-to-dig-deeper per metric.

~4h.
L11

Breadcrumb navigation · always visible "where am I"

Top-left breadcrumb: LLD > Creational > Factory Method > Learn Mode > §3 UML Reveal. Each segment clickable. Exact location context at all times. Currently breadcrumb exists but could be richer with section-level depth.

~3h polish.
L12

Mini-map intelligence · shows highlights, not just shapes

Current mini-map shows class silhouettes. Extend: highlighted/selected classes glow in mini-map. Lesson-scroll-section visible in mini-map as colored overlay. Mini-map becomes a navigation + attention map, not a preview.

~4h. Extend existing Minimap.tsx.
