Three design invariants this architecture enforces

1. Build mode stays untouched
Current LLDSidebar + LLDCanvas + LLDProperties + ContextualBottomTabs become the Build mode content. Zero regression risk for existing users.
2. Mode is a single piece of state
One field in ui-store: lldMode: "learn" | "build" | "drill". Persisted to localStorage. URL-reflectable via query param for sharing: ?mode=learn.
3. All 3 modes share one content source
Same DesignPattern / LLDProblem objects drive all 3 modes. Learn mode renders walkthrough + canvas highlight. Build renders full tools. Drill renders starter canvas + timer + grader. No content duplication.
