How users discover & pick a mode

Three scenarios: never-visited, returning user, deep-linked arrival. Each deserves a different landing experience.

Scenario 1 · First Visit — "I've never been here"

Strategy: Drop them into Learn mode (safest default) but show a dismissable welcome banner at the top offering all 3 paths. No forced gate. No onboarding wizard.

✓ Why banner, not full gate
Full-screen gates feel heavy. Banner respects the user's time — they can start learning while deciding. "Show, don't block."
✓ Why Learn as default
Most unknown users benefit from guidance. Advanced users (C/D) will immediately see the Build/Drill switcher and jump ship — and the banner gives them a 1-click path.

Scenario 3 · Deep Link — "Someone shared a URL with me"

Strategy: URL query params win over localStorage. This makes sharing predictable — the link creator's intent is honored.

# URL patterns supported:
/modules/lld
→ Last-used mode · or Learn if first visit
/modules/lld?mode=learn
→ Force Learn mode regardless of localStorage
/modules/lld?mode=learn&pattern=observer
→ Open Observer pattern in Learn mode · perfect for "check this out" links
/modules/lld?mode=drill&problem=parking-lot
→ Jump into Drill mode on Parking Lot · timer starts immediately
/modules/lld?mode=build&diagram=<id>
→ Open saved diagram in Build mode · for sharing your designs
Invariant: URL is authoritative. When user switches mode in UI, URL updates to reflect new mode (router.replace). Copy-paste URL = same view.

The decision flow, as code

Exact logic for "what mode do I render on page load?"

// On page load:
function resolveLLDMode(): LLDMode {
  // 1. URL wins if specified
  if (searchParams.mode) return searchParams.mode;

  // 2. localStorage preference
  const stored = uiStore.getState().lldMode;
  if (stored) return stored;

  // 3. First visit → Learn + show welcome banner
  uiStore.setState({ showLLDWelcomeBanner: true });
  return "learn";
}

Nudges over time (non-blocking)

Smart prompts that help users discover the other modes based on their activity. All dismissable, shown at most once per trigger.
