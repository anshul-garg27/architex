Q18 · What's shareable outside the app?

Your research file 11 lists "Multiplayer Battles" and "Peer Review" as P2 innovation features. But sharing is different — it's about getting external attention for the work you did inside the app. LinkedIn, Twitter, group chats.

A

No social · V1 is solo learning

No leaderboards, no shareable cards, no public profiles. Focus is pure learning. Add later if needed.

Pros: Cleanest scope. No moderation overhead.
Cons: Loses viral growth. Users can't show off progress — reduces organic acquisition.
B

Minimal · URL sharing only

Already designed in Q3/§4: deep-link URLs like /modules/lld?mode=learn&pattern=observer. User shares URL in Slack, recipient sees the same lesson. No images, no cards, no profiles.

Pros: Already free from Q3.
Cons: Plain URLs don't render nicely on Twitter/LinkedIn. Low signal.

D

Full social · leaderboards, multiplayer, comments

Every feature from the P2 innovation list. Full community platform.

Pros: Sticky community.
Cons: Moderation burden. Spam/abuse/trolling. Doubles scope. Requires active user base to not feel dead (chicken-egg).
💡 WHY C · SHARE, DON'T COMPETE
Interview prep pros don't want leaderboards. Competition is their day job. They want private improvement with public credentials.

OG cards are asymmetric viral. Every 95+ grade tweet = 3-10 new anonymous visitors. Zero effort to build that loop after infrastructure exists. "I solved Parking Lot in 24 minutes" with rich preview = perfect developer-Twitter content.

LinkedIn profile is career gold. "Certified in 22 GoF design patterns via Architex" is shareable. Users bragging = free marketing.

Gallery already exists. gallerySubmissions table, upvote logic, public submissions — all in the DB today. Just wire the UI, ship.
