Batch 2 of 5 · Motion & Interaction feel. Static UML is informational. Motion makes patterns feel alive. Each motion idea uses existing motion/react infrastructure already in your stack.

What makes the LLD module feel kinetic, not static?

Multi-select. These ideas layer motion on top of the solid visual base — subtle by default, respecting prefers-reduced-motion.

M1

Pattern signature animations · each pattern has a unique micro-motion

When you open a pattern, the UML classes perform a pattern-specific motion that embodies the pattern's essence. This becomes a visual mnemonic — you can recognize a pattern by the way its classes arrange.

🧘 Singleton — one class pulses gently at center, alone
📢 Observer — subject emits waves toward subscribers
🏭 Factory Method — Creator "produces" concrete products cascading down
⚡ Circuit Breaker — edge pulses, then snaps/blocks on failure threshold
🔧 Strategy — concrete strategies slide in/out of context slot
🧅 Decorator — wrappers nest around core class like rings
36 patterns × 3-5s animation. Can be AI-scripted via motion.dev keyframes. Plays once on pattern open; optional "replay" button. Respects reduced-motion. ~25-30h across all 36 (templates reduce per-pattern to ~45min each).
M2

Runtime particle flow · "watch the pattern actually run"

Inspired by your existing ParticleLayer in System Design. A "▶ Play" button on the canvas simulates the pattern at runtime — particles flow through the class graph representing method calls. Observer's notify() sends particles from subject to each subscriber simultaneously. Chain of Responsibility passes a single particle hop-by-hop until it stops. You see the pattern execute.

M3

Progressive class reveal · classes appear as lesson introduces them

Opening Learn mode: canvas starts near-empty. As you scroll through the lesson and it introduces each class, that class fades in + scales up on the canvas. By the time you finish the lesson, the complete UML has been built alongside you, not just presented.

Leverages Q7's highlightedClassIds — extend to visibleClassIds. Scroll position drives reveal. Motion: opacity 0→1, scale 0.9→1, 400ms ease-out with gentle stagger. ~6h.
M4

Magnetic hover · related classes glow together

Hover any class → it + all structurally-connected classes (via inheritance, composition, etc.) get a subtle shared glow. Non-connected classes dim to 40%. Reveals the pattern's structural "neighborhood" at a glance.

Your codebase already has "focus mode" (hover dims unrelated). This extends it: connected classes get an additional accent glow, not just kept-bright. ~3h polish on existing.
M5

Mode-switch choreography · crossfade + spatial transition

Clicking Learn → Build → Drill → Review currently swaps content instantly. Propose: 300ms transition — old content fades + slides out direction of target mode, new content fades in from opposite. Subtle but gives modes a spatial relationship in muscle memory (Learn feels "left of" Build, Drill "right of" Build).

Motion library handles this. Direction derived from mode index delta. Respects reduced-motion (fall back to instant crossfade). ~4h.
M6

Tinker unlock · ceremonial permission moment

Clicking "Tinker" in Learn mode should feel deliberate. Canvas briefly shimmers (gradient sweep across classes), class borders shift from solid to dashed (indicating editable state), toolbar slides down from top. Makes entering tinker mode feel like unlocking something, not just enabling a flag.

~3h for the shimmer + border transition + toolbar slide.
M7

Drill timer · heartbeat on last 5 minutes

Timer ticks silently for 20 min. At 5:00 remaining, it starts subtly pulsing — then at 1:00 remaining, the pulse speeds up slightly. Never hostile, just tonally urgent. Creates micro-tension at the right moments.

CSS keyframe + setInterval to set phase. ~2h.
M8

Ambient canvas breathing · subtle idle motion

When canvas is idle (no user interaction for 10s), existing ambient particles (already in your codebase!) drift more noticeably. When user touches canvas → particles recede. Signals "alive vs active" states without being distracting.

Tune existing lld-particle-drift animation opacity based on interaction timer. ~2h.

Multi-select M1-M8. Type choices or "all" / "none".
