Q4 · When is a pattern "mastered" — and what does that look like?

A user sees "Singleton" in the sidebar. What visual signal tells them their relationship with it? Done? Half-learned? Golden? This signal drives motivation and FSRS review cadence.

🎯 THE REAL STATES A USER CAN BE IN
○ Unseen
◐ Introduced (partial scroll)
◑ Completed (walkthrough done)
● Confident (1st-try checkpoints)
🎯 Applied (drill solved)
🔁 Retained (1 FSRS review)
⭐ Mastered (multi-review)
📉 Lapsing (FSRS decay)
8 possible states. But UI can't show 8 icons — user would be confused. We need to collapse these into a visible tier structure.
A

Binary · not-done vs done

One checkmark. No gradient. Pattern is either completed or not.

○ Singleton
✓ Factory Method
✓ Builder
○ Observer
Pros: Dead simple. Obvious at a glance.
Cons: Doesn't distinguish "scrolled through" from "truly knows it". No motivation to revisit. FSRS signal ignored at UI level.
B
⭐ MY RECOMMENDATION

3-tier · Introduced → Completed → Mastered · with decay

Three visible levels. Mastered decays back to Completed if no FSRS review passes in 90+ days — motivates ongoing review without punishing absence.

○  Singleton — not started
◐  Factory Method — Introduced
◉  Builder — Completed
★  Observer — Mastered
◉  Strategy — lapsing, review due
◐ Introduced: ≥50% scroll depth OR 1+ checkpoint attempted
◉ Completed: 100% scroll + ≥70% checkpoints correct (reveals count as wrong)
★ Mastered: Completed + drill passed (≥70% grade) + 1+ FSRS review retained
📉 Lapsing (not a tier, a warning): Completed/Mastered + FSRS says review is overdue (stability low). Shown separately.
Pros: Maps cleanly to 3 icon glyphs. Motivates both first-time completion and long-term retention. Drill integration baked in. FSRS lapsing surfaces as actionable nudge.
Cons: Needs 3 signals (scroll, drill, FSRS) to compute — slightly more DB reads. Manageable.
C

5-tier · full granularity

Seen / Introduced / Completed / Applied / Mastered. More precise but more icons = more cognitive load.

○ Seen · ◐ Introduced · ◑ Completed · 🎯 Applied · ★ Mastered
Pros: High fidelity. Nerdy users love this.
Cons: 5 states to explain. Users forget which icon means what. Too many.
D

Continuous 0-100% mastery score · FSRS-decaying

Each pattern shows a ring or number. Score computed from scroll, checkpoints, drill, FSRS stability. Decays over time without review.

⚪ Singleton · 0%
🟡 Factory Method · 45%
🟢 Builder · 78%
⭐ Observer · 94%
Pros: Scientifically precise. Percentage is universal.
Cons: Gameable feel ("must get to 100%"). Tuning the formula is painful. Doesn't celebrate milestones — no "aha moment" when you cross a threshold.
💡 WHY B WINS
3 tiers is the sweet spot — enough granularity to motivate (crossing Introduced → Completed is a celebration; Completed → Mastered is aspirational), but few enough to instantly parse. Each tier maps to a distinct glyph (◐ ◉ ★), visually scannable in a 36-item sidebar.

Decay is essential. Your product vision already bet on FSRS-5 as the spaced repetition engine (99.6% vs SM-2 superiority, per your README). Decay back to Completed when Mastered → lapsing is exactly the mechanism that creates the ongoing habit loop. Users who drift away get pulled back: "Your Observer mastery is lapsing — 2-min review?"

Drill integration is natural. "Mastered" requires drill pass → creates an organic bridge from Learn mode to Drill mode. Answers Q2's open question: "when should user graduate from Learn to Drill?" — when they want to turn Completed into Mastered.
COMPUTATION (server-side, cached)
function computeTier(progress, drillAttempts) {
  const scrolled = progress.metadata.scrollDepth >= 50;
  const completed = progress.completedAt !== null && progress.checkpointAccuracy >= 0.7;
  const drillPassed = drillAttempts.some(a => a.gradeScore >= 70);
  const reviewed = progress.reps >= 1 && progress.lapses <= progress.reps;
  const lapsing = progress.nextReviewAt < Date.now();

  if (completed && drillPassed && reviewed && !lapsing) return "mastered";
  if (completed) return "completed";
  if (scrolled) return "introduced";
  return "unseen";
}
