What happens when user switches mode mid-work?

Critical UX question — I'm proposing work-preserving transitions.

🎨 Build → 📖 Learn (user is tinkering, then wants to learn)
Build canvas state preserved in canvas-store (already persisted). Learn mode shows its own read-only reference canvas. When user switches back: their drawing is exactly where they left it. Toast: "Your work is saved — switching to Learn."
⏱️ Drill → 🎨 Build (mid-solve, wants to experiment)
Timer pauses. Canvas becomes editable with all their drill work intact. They can experiment freely. When they switch back to Drill: "Resume challenge? Timer: 23:41 remaining" with Resume / Give up / Submit now buttons.
📖 Learn → 📖 Learn (jumping between patterns)
Scroll position + checkpoint progress per pattern saved to progress-store. Return to same pattern later → land at same scroll position. "Resume where you left off" badge on the pattern card.
❓ Abandonment edge case
If user closes tab mid-drill → drill state cleared on next open (don't resume stale drills silently). BUT if they close mid-lesson → land exactly where they left off (learn is low-stakes).
