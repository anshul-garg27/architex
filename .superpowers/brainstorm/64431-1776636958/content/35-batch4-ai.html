A1

GitHub repo analysis · "what patterns are you already using?"

User connects a GitHub repo URL. Claude analyzes code, identifies which design patterns are already present, maps them to Architex's pattern catalog. Report: "We found Singleton in src/db/connection.ts:L12, Observer in src/events/bus.ts:L45, potential Factory Method opportunity in src/parsers/ParserFactory.ts (currently switch statement)." Learning feels personal, not abstract.

A2

AI-generated custom scenarios · "design a hotel booking system for Mars colonies"

User types any scenario prompt → Claude generates a fresh LLD problem with requirements, starter classes, hints, reference solution. Infinite problem pool. Pro-tier feature. Users can save their custom problems, share them to gallery.

A3

Socratic rubber duck · "explain your design out loud, AI probes"

After user builds in Drill mode, click "Explain out loud" → voice input (Whisper) or typed. Claude plays the interviewer, asks clarifying questions: "Why did you choose composition over inheritance here?" "What happens if Payment fails mid-transaction?" Tests your explanation skill — the actual interview skill, not just "can you draw it".

Claude Sonnet with architect persona (from existing socratic-tutor.ts). Whisper for voice. Can also use text. ~15h + voice stack.
A4

Anti-pattern detector · live in Build mode

As user builds in Build mode, Claude watches (debounced 10s of inactivity). If the canvas has an anti-pattern — god class, deep inheritance, circular dependency — a subtle ⚠️ badge appears on the offending class. Click → Claude explains why it's an anti-pattern + suggests the refactor.

A5

Pattern recommendation engine · "you might want Strategy here"

In Build mode, as user completes a recognizable shape (3+ classes implementing the same interface, a switch on type, etc.), Claude suggests: "Your design suggests Strategy — want to preview how that'd reshape this?" One click → shows a "ghost preview" of the refactored diagram.

Pattern-fingerprint heuristic first (fast, free) → Claude proposes specific pattern + preview (Sonnet). ~12h.
A6

Personalized study plan · "based on your weaknesses, here's 2 weeks"

User clicks "Generate study plan" → provides goals ("Meta interview in 3 weeks"). Claude analyzes their FSRS data, Mastered tier patterns, drill results. Returns a day-by-day 2-week plan: "Day 1: Learn Observer (30min). Day 2: Drill Parking Lot (30min). Day 3: Review flashcards (15min). ..." Editable, trackable.

Leverages progress-store data + Claude synthesis. ~15h incl. calendar-like UI.
A7

Company-style mock interviews · Amazon bar-raiser, Google leveling, Meta scope

Drill mode gets a preset: "Interview Company". Pick Amazon / Google / Meta / Stripe / Uber. Claude plays the interviewer with that company's known evaluation rubric — Amazon bar-raises for design simplicity, Google probes algorithmic complexity, Meta asks about scaling. Grade reflects company's specific bar.

Claude system prompts per-company (researched from Levels.fyi, Blind). ~10-12h + ongoing rubric refinement.
A8

AI-translated code · your pattern in Go, Rust, Kotlin, Swift, C++ on demand

Pattern page has TS/Py/Java columns (from C7). Add a dropdown "Translate to…" → Claude generates idiomatic version in Go/Rust/Kotlin/Swift/C++/Elixir. Cached per pattern-language combo. Covers every language an engineer might work in.

On-demand generation, cached. 36 patterns × 6 extra languages if fully generated = 216 files (but lazy). ~8h infra.
A9

AI flashcard generator · turn any lesson into Anki-style cards

After completing a lesson, one-click "Generate flashcards" → Claude creates 5-8 cards from that lesson's key concepts, formatted question/answer. Cards enter the FSRS review queue. Unlike templated flashcards, these are personalized per lesson content.

Haiku (cheap). Cached per lesson. ~6h.
