## Deviations from the plan

1. Folder name `build-mode/` instead of `build/` — the repo's root
   `.gitignore` has `build/` (no leading slash) which caught the
   original path. Renamed to avoid the collision and silent skip.

2. `computeGenericDagreLayout` instead of `computeDagreLayout` — the
   existing `dagre-layout.ts` exports `layoutDagre(UMLClass[], ...)`,
   not a React-Flow-compatible helper. Built a thin generic wrapper
   alongside so Build mode can layout arbitrary Node/Edge pairs.

3. Rate limiting wired through the existing `@/lib/security/rate-limiter`
   (token bucket, 20 bucket / 3-min refill = 20/hr) rather than a
   missing `@/lib/rate-limit` util. Per plan Open Questions §2 + the
   bootstrap instructions.

4. Seed runner not invoked (no DATABASE_URL in worktree). Seed function
   `seed(db)` exposed as default module export — production deploy
   runs it via `pnpm db:seed --module=lld-templates-library`.

5. Several React 19 React-Flow typed prop-shape quirks required
   adjusting plan snippets: `exportToJSON`/`DiagramJSON` usage for
   JSON export (plan's literal was the wrong DiagramJSON shape);
   `copyMermaidToClipboard(nodes, edges)` takes arrays directly.

6. `useAISuggestions` test is a logic-level test exercising the
   fetch contract rather than a React renderHook test, per the phase
   policy to avoid the React 19 act infra issue. Same for
   `useBuildKeyboardShortcuts` and `useAutoLayout` — all three use
   direct logic invocations.
